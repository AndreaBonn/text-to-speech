# Security

> **[Leggi in italiano](SECURITY.it.md)** | [Back to README](README.md)

This document describes the security mechanisms implemented in TTS Reader.
The application is designed for **single-user local use** and does not include
authentication, authorization, or rate limiting by design.

## HTTP Security Headers

All responses include security headers via `@app.after_request` in
[`app.py`](app.py):

| Header | Value | Purpose |
|--------|-------|---------|
| `X-Content-Type-Options` | `nosniff` | Prevents MIME-type sniffing |
| `X-Frame-Options` | `DENY` | Blocks iframe embedding (clickjacking) |
| `Referrer-Policy` | `strict-origin-when-cross-origin` | Limits referrer info on cross-origin requests |
| `Content-Security-Policy` | see below | Restricts resource loading sources |

### Content Security Policy

```
default-src 'self';
script-src 'self';
style-src 'self' https://fonts.googleapis.com;
font-src https://fonts.gstatic.com;
media-src 'self' blob:
```

- **No inline scripts or `eval()`** — only scripts served from the same origin
- **Media from `blob:`** — required for audio playback via `URL.createObjectURL()`
- **Fonts from Google Fonts** — only external resource allowed

## File Upload Validation

The `/api/load` endpoint ([`app.py`](app.py)) applies multiple layers of
validation before processing any uploaded file:

**Filename sanitization** (`_sanitize_filename`):
- Extracts basename via `PurePosixPath().name` — neutralizes path traversal
  (`../../../etc/passwd` becomes `passwd`)
- Regex whitelist `^[\w\-. ]+({extensions})$` — only word characters, hyphens,
  dots, spaces, and allowed extensions
- Rejects filenames with special characters, double extensions, null bytes

**Extension whitelist**:
- Only `.md`, `.txt`, `.epub`, `.docx`, `.html`, `.htm`, `.pdf` are accepted
- All other extensions return HTTP 400

**Size limit**:
- `MAX_CONTENT_LENGTH = 50 MB` — Flask rejects larger uploads with HTTP 413
- Prevents memory exhaustion and disk abuse

## Temporary File Handling

Uploaded files are written to OS-managed temp files with secure defaults:

```python
with tempfile.NamedTemporaryFile(suffix=ext, delete=False, mode="wb") as tmp:
    file.save(tmp)
try:
    paragraphs = engine.load_file(tmp_path)
finally:
    tmp_path.unlink(missing_ok=True)  # guaranteed cleanup
```

- **Unpredictable filenames** — generated by the OS, not derived from user input
- **Guaranteed cleanup** — `finally` block ensures deletion even on errors
- **No public directory exposure** — temp files live in `/tmp`, not in static/

The CLI ([`leggi.py`](leggi.py)) applies the same pattern for playback temp files.

## Input Parameter Validation

All API endpoints validate user-supplied parameters against explicit whitelists
before performing any processing:

- **Voice**: must be in `ALL_VOICES` (5 valid values)
- **Style**: must be in `ALL_STYLES` (4 valid values)
- **Paragraph index**: typed as `int` via Flask route (`/api/audio/<int:idx>`),
  bounds-checked inside a lock before array access

Invalid values return HTTP 400 with a descriptive error message.

## Output Encoding

The `/api/save` endpoint sanitizes the download filename:

```python
safe_name = "".join(c for c in f"{stem}.mp3" if c.isalnum() or c in ".-_ ")
encoded_name = quote(safe_name)
```

- Character filtering prevents header injection in `Content-Disposition`
- RFC 5987 encoding (`filename*=UTF-8''...`) for Unicode support
- Explicit `mimetype="audio/mpeg"` prevents MIME confusion

## Thread Safety

The TTS engine ([`tts_engine.py`](tts_engine.py)) uses a `threading.Lock` to
protect all shared mutable state:

- **Paragraph list** — read/write under lock
- **LRU cache** — access and eviction under lock
- **File loading** — clears cache atomically when a new file is loaded
- **Index bounds checking** — performed inside the lock before accessing paragraphs

The prefetch system uses a `ThreadPoolExecutor` for background synthesis,
with all cache writes going through the locked `_put_cache` method.

## Client-Side Security

The JavaScript player ([`static/player.js`](static/player.js)) follows these
practices:

- **`textContent` over `innerHTML`** — all dynamic text uses `textContent` to
  prevent DOM-based XSS
- **Blob memory management** — `URL.revokeObjectURL()` is called on every
  replaced blob to prevent memory leaks
- **No inline event handlers** — all events are bound programmatically

## Model Download Security

The Piper voice model download ([`synthesis.py`](synthesis.py)):

- Uses **HTTPS only** (HuggingFace URLs hardcoded in [`config.py`](config.py))
- Downloads in 64 KB chunks — no full-file memory allocation
- Writes to user home directory (`~/piper-voices/`) with standard OS permissions

## What Is NOT Implemented (By Design)

These mechanisms are intentionally absent because the application is
single-user and runs locally:

| Mechanism | Reason |
|-----------|--------|
| Authentication | Single-user, localhost only |
| CSRF tokens | No authenticated sessions to protect |
| Rate limiting | No multi-user abuse scenario |
| CORS | No cross-origin access needed |
| Database | No persistent storage — all data is in-memory |
| HTTPS/TLS | Expected to run behind localhost; use a reverse proxy for remote access |

## Reporting Vulnerabilities

If you discover a security vulnerability, please open an issue on
[GitHub](https://github.com/AndreaBonn/text-to-speech/issues) with the
label `security`.

## CI Security Checks

The project runs automated security audits via GitHub Actions:

- **[pip-audit](https://github.com/pypa/pip-audit)** — checks all Python
  dependencies against the OSV (Open Source Vulnerabilities) database on
  every push and pull request
- **[Ruff](https://github.com/astral-sh/ruff)** with
  [flake8-bandit](https://github.com/tylerwince/flake8-bandit) rules (`S` prefix) —
  static analysis for common security anti-patterns (hardcoded passwords,
  unsafe `eval`, insecure temp files, etc.)
